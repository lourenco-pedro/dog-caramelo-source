Managers

Referências:

Overview

O jogo irá conter os ManagerContainer's, onde cada um será reponsável por atualuzar um bloco importante do jogo. Sem eles o jogo não funciona. Cada manager tem sua configuração.

public class ManagerContainer<ManagerContainerType, ManagerConfigType> : SingletonMonoBehaviour<ManagerContainerType>
        where ManagerContainerType : MonoBehaviour
        where ManagerConfigType : MainConfig

Argumentos genéricos

A classe base apresenta dois argumentos genéricos.

  • ManagerContainerType: Irá definir qual tipo de Manager este será, este deverá ser um objeto Monobehaviour presente na cena
  • ManagerConfigType: Irá especificar qual configuração irá obecer. Este será um asset scriptable object

Exemplo:

CharacterManager : ManagerContainer<CharacterManager, CharacterManagerConfig>

API

Funções

Publicas

  • Initialize(): Chamado apenas uma vez, toda vez que o jogo é inicializado.

  • Run(): Esta função precisa ser chamada dentro da função Update da classe Game. Tem como objetivo rodar todo finalidade da Manager.

  • GetConfig(): Retorna

Variáveis

Privadas

  • bool _debug: Flag que irá definir se a Manager está em debug mode. Com ele ativado, por opção do programador, poderá logar messagens e executar ações que não pretende estar na versão final do jogo.
  • MainConfig _config: Field que contém a configuração. Definida no Inspector.
  • MainNavigationState _targetMainState: Estados de gameplay em que a Manager irá rodar. A classe Game contém o estado atual do jogo. Obs: A função Initialize roda independentemente do estado atual do jogo.

Propriedades

Privadas

  • bool Debug { get } : Se o jogo a Manager está em modo Debug.
  • MainConfig Config { get } : Configuração da Manager.
  • MainNavigationState TargetMainState { get }: Estados de jogo em que esta Manager irá rodar
